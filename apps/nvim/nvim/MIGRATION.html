⚠ いま「変・効かない」の正体は3つ

要対応 ① LSP が効かない

原因: LSP サーバ本体が未インストール（lua-language-server / pylsp / yaml-language-server が無い）。旧 vim は vim-lsp-settings が自動で入れていたが、新構成は起動を汚さないため自動インストールしない（存在するサーバだけ executable() で有効化）。

直し方 ▶ :Mason を開いて導入、または :MasonInstall lua-language-server pylsp yaml-language-server。導入後にファイルを開けば自動 attach。

無効化 ② space p 等のキーが効かない

原因: <Space>p（=<Leader>p）は CtrlP の :CtrlPMRU（最近使ったファイル）だった。CtrlP と Unite はレガシーのため廃止し fzf に集約したので、これらに紐づくマップが未割当になった。

キー旧動作状態
<Space>p:CtrlPMRU 最近使ったファイル無効
sf / sb / sB / sm / sa / srUnite 各種一覧無効
直し方 ▶ fzf へ再割当（例: <Space>p → :History）。希望あれば設定追加するのだ。

見た目変化 ③ statusline / tab の色が違う

原因: statusline を lightline(theme: seoul256) → lualine(theme: auto) に置換したため配色・構成が別物。tab 行も lightline が描いていたのが nvim デフォルトの tabline に戻った（showtabline=2 は維持だが装飾なし）。

直し方 ▶ lualine の theme を好みに固定 ＋ lualine の tabline 機能を有効化すれば近い見た目に戻せる。希望あれば調整するのだ。

仕組み — 自前 function をどう参照させたか

自前コマンド(:Clip/:SaveTemp/:Translate 等109個) と関数(GetGitRoot() 等80個) は全て 102_funcs.vim に定義済み。これを 書き直さず起動時に source することで nvim ネイティブのまま全部使えるようにしたのだ。

  • _G.src(name) … source ~/.vim/conf.d/<name> するだけのヘルパ（compat.lua 定義）
  • 各 conf.d 設定は先頭の if !g:plug.is_installed("name") | finish で自己ゲート
  • shim は lazy spec にその名前が有れば 1 を返す → 残すプラグインの設定だけが通る
  • 102_funcs 内の g:is_wsl/g:winclip 参照は全て関数内（実行時）→ source 時エラーなし。これら g: 変数は compat.lua が _G.is_* から派生

裏話 起動エラーの根治

旧 global.lua の autocmd ヘルパが引数キー groups/group 不一致で、イベント名(FileType/BufRead)の augroup を生成していた。これが source した autocmd FileType … の event 名を shadow し E216/E1155 を起こしていた。キー統一＋make_group 時のみ生成で根治。なお nvim --headless -es はエラーを握り潰すので検証では -es を外す必要があったのだ。

未対応・残置事項

挙動差 lazy 遅延ロードとキーマップの齟齬

旧 vim-plug は全プラグインを起動時一括ロード → 全キーが即有効。新 lazy は ft/cmd/event 遅延のため、source したキーマップが未ロードのプラグインコマンドを叩くと初回タイミングで効かない場合がある（例: <Space>s MarkdownPreview は markdown を開くまで未ロード）。起動エラーにはならない。

項目内容種別
<CR> 衝突nvim-cmp 確定 vs smartinput-endwise の括弧/end 補完の優先順位実機確認
denops 翻訳<Leader>tr 等は denops 非同期起動の初回待ちあり実機確認
100_base 未移植DisableLintInThisRepository / HighlightMixedIndent / RemoveAutoCommentAfterURL / highlight_notes軽微
taglistソースブラウザは未置換（レガシー）軽微

<CR> と denops は「起動時」でなく「キー実行時」の挙動のため headless 検証（エラー0）の範囲外なのだ。実機で対話確認が必要。

モード切替 (bin/nvim_switch)

  • lua モードへ: bin/nvim_switch（init.lua が無い状態で実行）
  • vim モードへ戻す: bin/nvim_switch（init.lua が有る状態で実行）
  • 現在モード確認: ls -la ~/.config/nvim/ → init.lua=lua / init.vim=vim

モード状態は ~/.config/nvim のローカル symlink。リポジトリにはコミットされないのだ。いつでも 1 コマンドで戻せる。
